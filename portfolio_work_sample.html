EPC Solar Engine

A Commercial-Grade Standalone Desktop Application for PV MCDA & Financial Analytics.

Project Overview

The EPC Solar Engine is an end-to-end Windows desktop application I architected to automate complex physical calculations and financial compliance for utility-scale solar projects. It replaces disjointed, manual spreadsheet workflows with a unified, data-driven ETL pipeline.

System Architecture

Backend / Data Pipeline

Engineered a high-performance Python backend using FastAPI and Pandas to ingest live unstructured EPD data via the Environdec API, computing physical module degradation and quantifying CBAM tax exposure dynamically.

Frontend / UI Analytics

Designed a highly responsive, interactive dashboard using React and Recharts to visualize sensitivity analysis via radar and tornado charts in real-time.

The Analytics Dashboard

The core UI interface was built to transform raw LCA and thermodynamic data into actionable, interactive visualizations for immediate decision-making.

Quantitative MCDA

Integrates Multi-Criteria Decision Analysis directly into the interface, allowing engineers to weigh Capital Expenditure (CAPEX) vs. Environmental Footprint instantly.

Interactive Data Viz

Utilizes dynamic SVG rendering to display complex radar and tornado charts that shift instantly as module parameters (e.g., temperature degradation) are tweaked.

The Problem Solved

EPC firms historically struggle to balance complex Life Cycle Assessment (LCA) compliance with financial profitability (LCOE/NPV). This engine automatically merges ESG compliance with predictive financial modeling, saving up to 60% of manual reporting time.

Automated Business Intelligence

A critical feature of the software is its ability to translate deep engineering metrics into plain-text commercial summaries for C-level executives.

The Executive Pitch Generator

Rather than forcing management to interpret raw data tables, the backend FastAPI engine automatically parses the calculated NPV, logistical scope 3 emissions, and CBAM tax exposure into a highly readable "Elevator Pitch."

This feature ensures that technical data is instantly actionable for commercial stakeholders, bridging the gap between rigorous Life Cycle Assessment (LCA) and real-world project finance.

Future Roadmap & Extensibility

The system architecture was deliberately decoupled to allow for future API integrations. The data ingestion pipeline is currently being optimized for Large Language Model (LLM) processing, and the physical models will soon expand to include Battery Energy Storage Systems (BESS) and Inverter Clipping Analytics.
